Notes:
- **Web sync:** Lead with the user outcome (one library, pick up where you left off). Subline on-frame can name the web app or “Deckbase on the web” if space allows.
- **MCP:** Many shoppers won’t know “MCP” yet—use the headline for the benefit (automation, Cursor, connected tools). Optional tiny subline: “Model Context Protocol” or “Works with Cursor & more” for technical buyers.

### Screenshot image ideas (pair each headline with real UI)

Prefer **authentic in-app captures** (lightly retouched). One clear story per slide; text overlay stays short so thumbnails stay legible.

| Headline cluster | What to show in-frame |
|------------------|------------------------|
| Turn notes and PDFs into cards | Source screen: PDF, article, or note with a visible **generate/import** action; optional split/stack: **document → generated card preview** so input→output reads instantly. |
| Generate AI flashcards fast / Create cards faster with AI | **AI generation** state: selection or prompt, then **several new cards** appearing (result or progress). Suggest speed with a tight crop, not busy copy. |
| Review with spaced repetition | **Study/review** UI: card front, reveal, **grade buttons** (Again/Hard/Good/Easy or your labels). Optional **due count** or “next review” so SR is obvious. |
| Remember more each week / Built for daily study | **Streak, stats, or today’s queue**: weekly activity, cards reviewed, simple chart, or calm **“Today’s review”** list—habit without noisy gamification. |
| Sync decks on web and mobile | **Two surfaces**: phone + laptop/browser mock showing the **same deck title or same card** on both; or one device with an explicit **synced / web** cue. Goal: one library everywhere. |
| Automate study with MCP workflows | **Split or bridge**: Cursor (or MCP client) with **Deckbase MCP** visible (tool call, server name, “add cards”) beside a **small phone** showing the resulting deck/cards. Optional arrow IDE→app. Subline: “Works with Cursor” if needed. |

**Order / polish**

- Put the **strongest outcome first** (materials→cards or review); place **web sync** and **MCP** where scrollers still see differentiation.
- **Thumbnail-safe:** large overlay type, high contrast, avoid tiny body text in the hero crop.
- **Series consistency:** same background, overlay font, and device frame choice (or all frameless) across the set.
