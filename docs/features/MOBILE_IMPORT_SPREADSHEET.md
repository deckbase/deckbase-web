# Import Spreadsheet – Mobile Implementation Guide

This doc describes how to implement the **Import spreadsheet** feature on mobile so it matches the web flow: load file → select rows → map columns to template blocks → run import (AI quiz generation, create cards, optional TTS per card).

**Related:** [File to AI Cards](./FILE_TO_AI_CARDS_FEATURE.md) (different flow: single file → AI generates cards in one shot). [Add Cards with AI – Mobile](./MOBILE_ADD_CARDS_WITH_AI.md) (card payload shape and add endpoint). [ElevenLabs TTS – Mobile](./ELEVENLABS_API_MOBILE.md).

---

## 1. Overview

| Item | Description |
|------|-------------|
| **Input** | CSV, Excel (.xlsx, .xls), or Anki (.apkg). User selects file, then which rows to import, then maps columns to template blocks. |
| **Output** | Cards created in the deck via Firestore (same as web). Quiz content comes from Claude; audio “Use AI” uses row text + ElevenLabs TTS. |
| **Auth** | Same as rest of mobile: `X-API-Key: <DECKBASE_API_KEY>`. For `import-ai-blocks`, production also expects Firebase ID token (Bearer) or the same API key. Pro/VIP required for AI and for Excel/APKG. |

**Flow (4 steps):**

1. **Load file** – Pick CSV / Excel / APKG; parse to headers + rows.
2. **Select rows** – Choose which rows to import (all or a subset).
3. **Map columns** – Map each template block to a spreadsheet column, or “Generate with AI” for quiz/audio blocks.
4. **Importing** – Run quiz AI (batched), then create cards and optionally generate audio per card.

---

## 2. Steps (match web)

Use the same step enum so behavior is consistent:

```ts
const ImportStep = {
  loadFile: 1,
  selectRows: 2,
  mapColumns: 3,
  importing: 4,
};
```

- **loadFile:** File picker; after parse, go to **selectRows**.
- **selectRows:** Show table (or list); user selects rows; next → **mapColumns**.
- **mapColumns:** User picks template and maps columns to blocks (and which blocks use “Generate with AI”); next → **importing**.
- **importing:** Run API + Firestore + TTS; show progress; on done show success and “Done”.

---

## 3. Parsed file data shape

After parsing CSV/Excel/APKG, use a single structure (web uses `SpreadsheetData`; mobile can use a plain object):

| Field | Type | Description |
|-------|------|-------------|
| `headers` | `string[]` | First row: column names (e.g. `["Term", "Definition", "Example"]`). |
| `rows` | `string[][]` | Data rows; each row is an array of cell strings. |
| `fileName` | `string` | Original file name (e.g. `"vocab.xlsx"`). |
| `sheetName` | `string` | (Optional) Excel sheet name; default `"Sheet1"`. |
| `rowCount` | `number` | `rows.length`. |
| `columnCount` | `number` | `headers.length`. |

- **CSV:** Parse with a CSV library; first line = headers, rest = rows. Handle quoted commas and newlines.
- **Excel:** Use a native or cross-platform lib (e.g. xlsx / SheetJS, or platform-specific APIs). First row = headers; remaining rows = data; drop completely empty rows.
- **APKG:** Use existing APKG parser if you have one; output the same `{ headers, rows, fileName }` shape (e.g. “Front” / “Back” columns).

Web reference: `utils/spreadsheetParser.js` (`parseFile`, `parseBytes`, `SpreadsheetData`).

## 4. Column mappings

User maps **spreadsheet columns** to **template blocks**. Each mapping:

| Field | Type | Description |
|-------|------|-------------|
| `columnIndex` | `number` | 0-based column index in `headers`/`rows`. |
| `blockId` | `string` | Template block ID. |
| `columnName` | `string` | Display: `headers[columnIndex]` or `"Column N"`. |
| `blockLabel` | `string` | Template block label (e.g. “Header 1”, “Hidden text”). |
| `generateWithAI` | `boolean` | If true, content for this block is generated by AI (quiz) or TTS (audio), not from the column. |

Rules:

- **Text blocks:** Map to a column; value = `row[columnIndex]`. No `generateWithAI`.
- **Quiz blocks:** Map to a column for context; set `generateWithAI: true`. Quiz content (question, options, correct answers) comes from **Claude** via `POST /api/cards/import-ai-blocks` (see below).
- **Audio blocks:** Map to a column for “source text”; set `generateWithAI: true`. Audio is generated with **ElevenLabs TTS** from the row’s “main text” (see “Main text per row”) after the card is created.

At least one block must be mapped to proceed to importing.

## 5. APIs and Firestore

### 5.1 Quiz generation (batched)

**POST** `/api/cards/import-ai-blocks`

- **Auth (production):** `Authorization: Bearer <Firebase ID token>` (same as web). Get the ID token from Firebase Auth (e.g. `user.getIdToken()`). The backend does not use `X-API-Key` for this route.
- **Body (JSON):**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `extractedContent` | string | Yes | TSV: first line = headers (tab-joined), then one line per row (tab-joined). Max length ~50k chars. |
| `deckId` | string | Yes | Deck ID. |
| `templateId` | string | Yes | Template ID. |
| `uid` | string | Yes | Current user ID. |
| `maxCards` | number | No | Max cards per request (1–30). Default 15. Should match chunk size. |
| `blockIds` | string[] | Yes | **Quiz block IDs only** (no audio). Claude fills question/options/correctAnswers for these. |

- **Batching:** If selected rows > 30, split into chunks of 30. Call this API once per chunk. Build `extractedContent` as:

  ```
  headers.join("\t") + "\n" + chunkRows.map(r => r.join("\t")).join("\n")
  ```

  Truncate to 50,000 characters if needed.

- **Response (200):**

  ```json
  { "cards": [ { "templateId", "blocksSnapshot", "values", "mainBlockId", "subBlockId" }, ... ] }
  ```

  Each item in `cards` corresponds to one row in the chunk. Use `blocksSnapshot` and `values` when building the card for that row (merge with template and column-derived values).

- **Errors:** 400 (missing/invalid body, empty `blockIds`), 401 (auth), 403 (not Pro/VIP or limit), 404 (template not found), 502 (AI invalid JSON), 503 (ANTHROPIC_API_KEY not set).

### 5.2 Create card (Firestore)

Same as web. Use your existing “create card” API or direct Firestore API.

**createCard(uid, deckId, blocksSnapshot, values, templateId, mainBlockId, subBlockId)**

- `blocksSnapshot`: Array of block objects (blockId, type, label, required, configJson). For rows with “Generate with AI” quiz, merge in the corresponding `blocksSnapshot` from the import-ai-blocks response for that row index.
- `values`: Array of `{ blockId, type, text }` (and for audio block, `mediaIds` when you have TTS). For mapped columns, `text = row[columnIndex]`; for AI quiz, use the `values` from the API for that row; for audio “Use AI”, start with main text, then replace with `mediaIds` after TTS.
- Returns created card (includes `cardId`). Use `cardId` for `updateCard` when attaching TTS.

Reference: web `utils/firestore.js` – `createCard`, `updateCard`, `transformBlockToFirestore`, `transformValueToFirestore`.

### 5.3 Update card (after TTS)

When a row has audio “Use AI” and you have generated audio:

**updateCard(uid, cardId, deckId, values, blocksSnapshot, mainBlockId, subBlockId)**

- `values`: Same as at create, but the value for the audio block has `mediaIds: [mediaId]` (from upload).
- `blocksSnapshot`: Same as at create (no change for audio block config).

### 5.4 Text-to-speech (ElevenLabs)

**POST** `/api/elevenlabs/text-to-speech`  
**Headers:** `X-API-Key: <DECKBASE_API_KEY>`, `Content-Type: application/json`  
**Body:** `{ "text": "<main text for row>", "voice_id": "<optional from template audio block config>" }`  
**Response (200):** Binary `audio/mpeg` (MP3).

- Call this **after** creating the card, for each row that has an audio block with “Generate with AI” and non-empty main text.
- Upload the MP3 to your storage (same as web: Firebase Storage under `users/{uid}/media/{mediaId}.mp3`), create media doc in Firestore, then call `updateCard` with the audio block’s `values` entry set to `mediaIds: [mediaId]`.
- Voice: From template’s audio block `configJson.defaultVoiceId` (or `default_voice_id`). See `lib/audio-block-config.js` (`parseAudioBlockConfig`).

See [ELEVENLABS_API_MOBILE.md](./ELEVENLABS_API_MOBILE.md) for auth and errors.

### 5.5 Main text per row (for audio “Use AI”)

For TTS, you need one string per row. Use in order:

1. The value from the column mapped to the template’s **main block** (`mainBlockId`), if that column is mapped and in range.
2. Otherwise the first non-empty value from any mapped column for that row.

Use this string as `text` in the TTS request. If it’s empty, skip TTS for that row (no call, no failure).

## 6. Import run (order of operations)

1. **Prepare:** Get selected rows `rows = getRowsToImport()` (all or selected subset). Get `columnMappings`, `selectedTemplate`, `deckId`, `uid`.
2. **Quiz phase (if any “Generate with AI” quiz blocks):**
   - Collect quiz block IDs: `quizBlockIds = mappings where generateWithAI && block is quiz`.
   - Chunk rows into chunks of 30. For each chunk:
     - Build `extractedContent` (TSV: headers + chunk rows).
     - `POST /api/cards/import-ai-blocks` with `blockIds: quizBlockIds`, `maxCards: chunk.length`, etc.
     - Append returned `cards` to an `aiCards` array (index by global row index: chunk offset + index in chunk).
   - Update progress: e.g. “Generating quiz/audio with AI…”, “Batch X of Y”, and a progress bar (see below).
3. **Card phase:**
   - For each row index `i`:
     - Build `values`: from column mappings (column value or from `aiCards[i]` for quiz, or main text for audio “Use AI”).
     - Build `blocksSnapshot`: template blocks with quiz config merged from `aiCards[i].blocksSnapshot` when “Generate with AI” quiz.
     - Skip row if no text and no AI-generated quiz content.
     - Call **createCard**(…). Get `cardId`.
     - If this row has audio “Use AI” and main text:
       - Set “Generating audio…” (or similar).
       - Call TTS with main text (and optional voice_id).
       - Upload audio to storage, get `mediaId`.
       - Call **updateCard** with the audio block’s value having `mediaIds: [mediaId]`.
       - Track “audio completed” / “audio failed” for UI.
     - Update progress: “Importing cards…”, “X of Y cards imported”, and optional “Audio: N completed, M failed”.
4. **Done:** Clear progress state; show “Import Complete! Successfully imported N cards” and optional “Audio: N completed, M failed”. Offer “Done” to close.

## 7. Progress UI (match web)

- **Unified progress (0–1):**  
  - Total steps = (number of quiz batches) + (number of rows).  
  - During quiz: `progress = (batchIndex + 1) / totalSteps`.  
  - During cards: `progress = (chunks.length + importedCount) / totalSteps`.

- **Phase 1 – Quiz generation:**
  - Title: “Generating quiz/audio with AI…”
  - Subtitle: “Batch X of Y” and a progress bar (or circular progress) using the unified progress.

- **Phase 2 – Card import:**
  - Title: “Importing cards…”
  - Subtitle: “X of Y cards imported”.
  - If currently generating TTS: “Generating audio…”
  - If any audio was attempted: “Audio: N completed, M failed” (M in red if > 0).

- **Completion:**
  - “Import Complete!” / “Successfully imported N cards”.
  - If any audio: “Audio: N completed, M failed” below.

Reset progress (and audio counts) when the user leaves the flow or starts a new import.

---

## 8. Block types (quiz vs audio)

- **Quiz blocks:** `quizSingleSelect`, `quizMultiSelect`, `quizTextAnswer` (or numeric 8, 9, 10). Only these go in `blockIds` for `import-ai-blocks`. Claude returns `blocksSnapshot` with `configJson` containing `question`, `options`, `correctAnswers`, etc.
- **Audio blocks:** Type “audio” (or your app’s constant). Do **not** send to `import-ai-blocks`. “Use AI” means: use main text for the row and generate audio via ElevenLabs, then attach `mediaIds` on the card.

---

## 9. Pro/VIP and limits

- **Excel / APKG import:** In production, require Pro/VIP (same as web). CSV can be free.
- **import-ai-blocks:** Requires Pro/VIP in production; also subject to monthly AI generation limits.
- **TTS:** Pro/VIP and monthly character limit (see ElevenLabs doc). On 403 or TTS failure, count as “audio failed” and continue; do not block the rest of the import.

---

## 10. Errors and edge cases

| Case | Handling |
|------|----------|
| import-ai-blocks 401/403 | Show error; stop import. |
| import-ai-blocks 502/503 | Show “AI generation failed”; stop. |
| TTS 4xx/5xx or throw | Log; increment “audio failed”; continue with next row. |
| createCard / updateCard fail | Show error; stop or retry per your policy. |
| Row with no text and no quiz | Skip row (do not create card). |
| Empty main text for audio “Use AI” | Skip TTS for that row; no failure. |
