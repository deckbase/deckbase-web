# AI-generated images (fal.ai) — feasibility and design notes

This document assesses adding **AI image generation** for image blocks, in the same spirit as **AI audio** (ElevenLabs TTS): server-side generation behind auth and plan limits, then attach generated bytes to the card via existing **Firebase Storage + `mediaIds`** flows.

**Status:** Product decisions **locked** (see below). Implementation may follow this spec.

## Locked product decisions

| Topic | Decision |
|-------|----------|
| **Model choice** | User picks from a **small curated list** in the UI (server validates `model_id` against the same allowlist). No arbitrary fal endpoints from clients. |
| **Entitlement** | **Basic and Pro** (same pattern as TTS: Free = no access). VIP/Pro tier uses Pro limits. |
| **Quota** | **Credit-based monthly pool** with **per-model weights** (expensive models consume more credits). Track **usage per `model_id`** (counts + credits consumed). Limits: **Basic 40 / Pro 100 credits/mo** — see [Usage tracking & quota (per-model credits)](#usage-tracking--quota-per-model-credits) and [Quota rationale](#quota-rationale-monthly-limits). |
| **Marketing copy** | Update **subscription dashboard** and **public pricing** so tiers list **AI image credits/month** (40 Basic / 100 Pro) and that **cost varies by model**. |
| **Scope (one release)** | **Web card editor** + **import / “Use AI”** (deck page) + **mobile** (same API as TTS auth pattern) + **MCP** (`create_card` / attach flows as designed in implementation). |
| **Reference image** | Optional **reference image** for generation: user can **upload** a file or **pick from generated history** (see [Reference image](#reference-image-upload--generated-history)). |
| **Style prompt library** | **Curated preset snippets** with **tags** (subject e.g. STEM, vocabulary; style e.g. Ghibli, anime) for filtering. Users **append** snippets to their prompt. **Basic and Pro only**. See [Style prompt library](#style-prompt-library-curated-presets). |

## Quota rationale (monthly limits)

Variable cost is **per successful run** and **differs by model** ([fal.ai pricing snapshot](#falai-pricing-snapshot-verified-on-model-pages)). Deckbase uses **credit points** per month, not a flat “image count,” so a Nano Banana Pro run cannot consume the same allowance as a FLUX Schnell run.

| Plan | Price (mo) | Limit (**credits/mo**) | Planning note (API variable only) |
|------|------------|------------------------|-----------------------------------|
| Basic | $5.99 | **40** | If users average **~1 credit/run** (Schnell/Grok-class), ~**$0.035/run** effective → ~$1.40/mo at full Basic pool. |
| Pro | $11.99 | **100** | Same assumption → ~$3.50/mo at full pool. |

**Notes:**

- **Credits are not “one image = one credit.”** Each `model_id` (including reference/edit endpoints) has a **weight** in the server **`CREDIT_COST_BY_MODEL`** map — see [Credit cost table](#credit-cost-table-server-constant).
- Margins are **directional**, same spirit as the TTS table in `docs/subscription/PRICING.md`. **Reconcile** against fal billing; **tune weights** if fal changes prices.
- **Separate from `aiGenerations`** so LLM card gen and image gen do not share the same pool.

## Usage tracking & quota (per-model credits)

### Why

- **Fair quota:** Users who mostly pick **cheap** models (e.g. Schnell) get more generations than users who only pick **expensive** ones (e.g. Nano Banana Pro), for the **same monthly cost** to Deckbase.
- **Analytics:** Per-model counts show which endpoints drive cost and support (“everyone uses Pro model X”).
- **Enforcement:** `checkImageGenerationLimit(uid, resolvedModelId)` compares **`imageCreditsUsed + cost(resolvedModelId)`** to the tier **credit cap**.

### Firestore shape (monthly doc `users/{uid}/usage/{YYYY-MM}`)

| Field | Type | Purpose |
|-------|------|---------|
| `imageCreditsUsed` | number | Sum of credit weights consumed this month (increment by cost on each success). |
| `imageGenerationsByModel` | map (optional) | `model_id` → **count** of successful calls (integers). For dashboards, support, cost attribution. |
| `imageGenerations` | number (optional) | **Total successful image API runs** (raw count), if you want a simple “# of generations” metric alongside credits. |

**Implementation:** On each successful fal call, atomically **`increment`** `imageCreditsUsed` by `CREDIT_COST_BY_MODEL[resolvedModelId]` and **`increment`** `imageGenerationsByModel[resolvedModelId]` (merge). Use **transaction** or **`FieldValue.increment`** to avoid races.

**Resolved model id:** The **actual** fal endpoint invoked — e.g. `fal-ai/flux-pro/kontext` when user chose Schnell + reference, not `fal-ai/flux/schnell`.

### Credit cost table (server constant)

Integer **credit weights** (tune to fal pricing; revisited when fal changes rates). Baseline: **1 credit ≈ one “typical cheap” T2I** (~Schnell-class $/run).

**Text-to-image (master list)**

| `model_id` | Credits per success |
|------------|---------------------|
| `fal-ai/flux/schnell` | **1** |
| `fal-ai/flux/dev` | **1** |
| `fal-ai/nano-banana-2` | **3** |
| `fal-ai/qwen-image` | **1** |
| `fal-ai/recraft/v3/text-to-image` | **2** |
| `fal-ai/nano-banana-pro` | **5** |
| `fal-ai/stable-diffusion-v35-large` | **2** |
| `fal-ai/fast-sdxl` | **1** |
| `xai/grok-imagine-image` | **1** |

**Reference / edit (resolved endpoint)**

| `model_id` | Credits per success |
|------------|---------------------|
| `fal-ai/flux-pro/kontext` | **2** |
| `fal-ai/nano-banana-2/edit` | **3** |
| `fal-ai/nano-banana-pro/edit` | **5** |
| `fal-ai/stable-diffusion-v3-medium/image-to-image` | **2** |
| `fal-ai/fast-sdxl/image-to-image` | **1** |

**Rules**

1. **`checkImageGenerationLimit(uid, resolvedModelId)`** — load `imageCreditsUsed`, tier cap (**40 / 100**), `cost = CREDIT_COST_BY_MODEL[resolvedModelId]`; if `used + cost > cap`, **403** with message showing remaining credits.
2. **`incrementImageUsage(uid, resolvedModelId)`** — increment `imageCreditsUsed` by `cost`, bump `imageGenerationsByModel[resolvedModelId]`, optionally `imageGenerations` total.
3. **GET usage API** (if any) — return `imageCreditsUsed`, `imageCreditsLimit`, **`breakdownByModel`** from `imageGenerationsByModel` for UI (“You used 12 credits on Schnell, 15 on Nano Banana 2…”).

### UX copy

- Prefer **“AI image credits”** in subscription and settings rather than “40 images” so users understand Pro-tier models cost more.
- Show **remaining credits** before generate; optional **estimated cost** next to model picker (“This model uses 5 credits”).

## Style prompt library (curated presets)

**Product:** A **library of predefined style modifiers**—short text snippets users can select to **combine** with their own subject prompt (e.g. user types “a fox reading a book” and picks **“soft watercolor children’s book”** from the library → final prompt is merged for fal). UX is inspired by prompt marketplaces such as [PromptBase](https://promptbase.com/) (browse by style/mood), but **Deckbase ships its own original copy**; **do not** scrape or resell third-party marketplace prompts (copyright / ToS).

### Access control

| User | Library |
|------|---------|
| **Free** | No access — show upgrade CTA (same tier gate as AI image generation). |
| **Basic / Pro / VIP** | Full library list + append behavior in the image **Generate with AI** UI. |

**Server:** `GET /api/ai/image-style-prompts` (name TBD) returns the catalog **only** after `isBasicOrProOrVip(uid)` (or equivalent) in production. **Query params** (optional): `?tag=...` and/or `?tagGroup=...` to filter (see [Tag taxonomy](#tag-taxonomy)). **Do not** ship the full list in static client bundles for unauthenticated users if copy is considered premium IP—use authenticated fetch or edge config gated by session.

### Tag taxonomy

Each entry carries one or more **tags** from a **controlled vocabulary** (enforced in code when authoring the JSON; API validates unknown tags). Tags power **filters** in the UI (e.g. “Language learning”, “Physics”, “Anime style”) so users can find presets for flashcard use cases and visual styles.

**Suggested tag groups** (same tag string can appear in filters under multiple groups):

| Group | Purpose | Example tag values |
|-------|---------|-------------------|
| **Learning & subject** | What the card is *for* | `language-learning`, `vocabulary`, `stem`, `physics`, `chemistry`, `biology`, `mathematics`, `history`, `geography`, `literature`, `exam-prep` |
| **Visual style & medium** | How the image should *look* | `ghibli`, `realistic`, `photographic`, `abstract`, `anime`, `watercolor`, `flat-illustration`, `minimal`, `line-art`, `diagram`, `infographic` |
| **Mood & tone** (optional) | Atmosphere | `creepy`, `whimsical`, `serious`, `playful`, `cinematic` |

**Rules**

- Use **lowercase kebab-case** for machine tags (`language-learning`, not `"Language Learning"`). Display labels are **localized** in the UI.
- A preset can have **multiple tags** (e.g. `["vocabulary", "flat-illustration", "playful"]`) so it appears under “Vocabulary” and “Flat illustration”.
- **Filter logic:** `GET ?tag=physics` returns entries where `tags` contains `physics`. `?tags=physics,anime` could mean AND or OR—pick one and document (OR is friendlier for discovery).

### Data shape (example)

Static JSON in repo or Firestore admin collection; each entry:

| Field | Purpose |
|-------|---------|
| `id` | Stable key (`ghibli-soft`, `chem-molecule-flat`, …). |
| `label` | Short UI title (“Ghibli-like”, “Chemistry — flat diagram”). |
| `snippet` | Text **appended** (or prefixed—pick one convention) to the user prompt. |
| `tags` | **string[]** — controlled vocabulary (see [Tag taxonomy](#tag-taxonomy)). |
| `description` | One-line helper text in the picker. |
| `tagGroups` | Optional **derived** UI hint: `["learning", "style"]` for layout only—do not duplicate truth; `tags` is canonical. |

**Merge rule (recommended):** `finalPrompt = userPrompt.trim() + ", " + snippet` (or newline-separated). Document in UI: “Adds style: …”

### UX

- **Browse:** Filter chips or tabs by **tag** / **tag group** (e.g. “Subject” vs “Style”)—similar discovery to [PromptBase](https://promptbase.com/) categories, without leaving Deckbase.
- **Search:** Optional text search over `label` + `description` + tag display names.
- **Multi-select:** Optional v2; v1 can be **one style snippet** per generation to keep prompts readable.
- **Preview:** Show merged prompt in a read-only textarea before **Generate** so users can edit.

### Implementation notes

- **i18n:** If the app supports multiple locales, either translate labels/descriptions or keep snippets English-only for fal and translate only UI chrome.
- **Versioning:** Bump library version when snippets change so support can reference “library v3”.
- **MCP / API:** Optional `style_prompt_id` on generate-image API; server resolves snippet and merges—**never** trust raw long style text from client without allowlist.

### Checklist (library-specific)

1. Author and legally clear **original** snippets (or license explicitly if any third-party text is used); assign **`tags`** from the controlled vocabulary.
2. Implement gated **GET** list (optional **`?tag=`**) + merge in **POST** generate route.
3. Card editor UI: style picker with **tag filters** + merge preview for subscribed users only.

| | **Pros** | **Cons** |

## Reference image (upload + generated history)

Users should be able to attach an **optional reference image** when generating, in addition to the text prompt. Two sources:

| Source | Behavior |
|--------|----------|
| **Upload** | User picks a file from disk (same patterns as existing image block upload: validate type/size, optional downscale client-side before API). |
| **Generated history** | User picks from **recent AI-generated images** they created in Deckbase (not arbitrary web URLs). |

### Generated history — scope (recommended)

| Scope | Purpose |
|-------|---------|
| **This card / session** | Quick iteration: thumbnails of outputs generated in the current editor session (in-memory + optional short-lived cache). |
| **Account recent** | Persisted list: last **N** successful AI image generations for `uid` (e.g. `users/{uid}/aiImageHistory` or entries under `usage` with metadata: `thumbnailUrl`, `fullMediaId`, `createdAt`, `prompt`, `model_id`). **N** small (e.g. 20–50) with FIFO eviction. |

**Privacy / security:** History entries must belong to the authenticated user. The API must **not** accept arbitrary external URLs for reference unless the server verifies the file lives under **`users/{uid}/`** in Storage or passes through a verified signed URL for that user’s media.

### T2I selection → reference endpoint routing

When the user picks a **text-to-image** model from the [master list](#master-list-role) **and** attaches a reference, the backend **does not** call the T2I id—it maps to a **reference row** below. The UI can still show the **same brand** (e.g. “Nano Banana 2”) while the server calls the `/edit` endpoint.

| User’s T2I choice (`model_id`) | Call this reference `model_id` instead | Notes |

### API shape (sketch)

- **Option A — multipart:** `POST` with `prompt`, `model_id`, optional `reference` file field.
- **Option B — JSON + Storage:** Body includes `prompt`, `model_id`, optional `reference_media_id` (Firestore media owned by user) or `reference_storage_path`; server loads bytes from Storage, then calls fal.

**Quota:** Each successful generation increments **`imageCreditsUsed`** by the **credit weight** for the **resolved** `model_id` and updates **`imageGenerationsByModel`** — see [Usage tracking & quota](#usage-tracking--quota-per-model-credits).

### Pros and cons (reference image)

| | **Pros** | **Cons** |
|---|----------|----------|
| **Upload** | Works with any user asset (sketch, photo, style reference). | Larger payloads; must scan/validate; Storage bandwidth if server pulls user files. |
| **Generated history** | Fast reuse of last outputs; great for iterative refinement without re-uploading. | Requires **history storage + UI** (thumbnails, pagination); retention policy; slightly more Firestore/Storage cost. |
| **Overall** | Matches user expectations from other AI image tools (style transfer, “more like this”). | **Two code paths** (T2I vs reference-conditioned); more models to maintain; higher support load. |

## How AI audio works today (reference pattern)

Understanding this flow is the fastest way to design AI images.

1. **API:** `POST /api/elevenlabs/text-to-speech` (`app/api/elevenlabs/text-to-speech/route.js`)
   - Uses `requireElevenLabsAuth` (`lib/elevenlabs-auth.js`): in production, Bearer (web) or `X-API-Key` (mobile); dev often skips strict auth.
   - Enforces **Pro/Basic/VIP** and **monthly TTS character limits** via `checkTTSLimit` / `incrementTTSChars` (`lib/usage-limits.js`).

2. **Server generation:** Calls ElevenLabs HTTP API with `ELEVENLABS_API_KEY`; returns `audio/mpeg`.

3. **Client (card editor):** `handleGenerateAudio` in `app/dashboard/deck/[deckId]/card/[cardId]/page.js`
   - `fetch` → `blob` → `File` → `checkStorageBeforeUpload` → `uploadAudio` → updates `values[blockId].mediaIds` → `persistCard`.

4. **Reuse for images:** Swap ElevenLabs for **fal.ai** in a **new route** (e.g. `POST /api/fal/text-to-image` or `/api/ai/generate-image`). After generation, use **`uploadImage`** instead of `uploadAudio`, reusing storage checks and persist logic.

## fal.ai integration

- **Docs / models:** Model list and schemas live on [fal.ai models](https://fal.ai/models) (e.g. `fal-ai/flux/dev`, `fal-ai/flux/schnell`, or other `fal-ai/*` endpoints). Each model has its own **input fields** (prompt, size, steps, etc.).
- **Client:** `@fal-ai/client` supports `fal.config({ credentials: "FAL_KEY" })`, then `fal.subscribe("<model-id>", { input: { ... } })` for queue-based runs (typical for image gen).
- **Env:** Server should set **`FAL_KEY`** (never expose to the browser). The existing client package is **not** wired in application code yet—only declared as a dependency.

## Target architecture (implementation)

1. **`lib/usage-limits.js`**
   - Constants: **`IMAGE_CREDIT_LIMIT_BASIC = 40`**, **`IMAGE_CREDIT_LIMIT_PRO = 100`** (monthly **credit** pools).
   - **`CREDIT_COST_BY_MODEL`**: map from resolved `model_id` → integer credits ([table](#credit-cost-table-server-constant)).
   - `getMonthlyUsage`: include **`imageCreditsUsed`** (default 0), optional **`imageGenerationsByModel`** map.
   - **`checkImageGenerationLimit(uid, resolvedModelId)`** — compares `used + cost` to tier cap.
   - **`incrementImageUsage(uid, resolvedModelId)`** — increment credits + per-model counts (and optional total `imageGenerations`).

2. **New API route** (e.g. `POST /api/fal/text-to-image` or `/api/ai/generate-image`)
   - Auth: same as TTS — **`requireElevenLabsAuth`** (or shared helper) + production **`isBasicOrProOrVip`** + **`checkImageGenerationLimit(uid, resolvedModelId)`** **before** call; **`incrementImageUsage`** after success.
   - Body: `prompt`, **`model_id`** (must be curated allowlist), optional size/seed; optional **`reference_media_id`** / upload / **reference** file per [Reference image](#reference-image-upload--generated-history). If reference present, use a **reference-capable** model id or a server-side mapping to img2img/kontext endpoints.
   - Server: `fal.subscribe(model_id, { input })` → fetch image URL or bytes → respond as image binary or JSON with short-lived URL + type; client uploads via existing **`uploadImage`** path.
   - **Storage:** `checkStorageBeforeUpload` before persisting.
   - **On success:** append to **generated history** for the user (thumbnail + `mediaId` + metadata) for “pick from history” next time.

3. **Web card editor** — `BlockEditor` `case "image"`: **Generate with AI** (prompt, model picker, optional **reference** via upload + **history strip**, Basic/Pro upsell, loading), same entitlement pattern as audio TTS.

4. **Import / deck “Use AI”** — extend row/column logic so image blocks can receive a **prompt** (or mapped column → prompt) and call the same API + attach `mediaIds`; enforce **credit** quota per successful generation using resolved `model_id`.

5. **Mobile** — same route as web; auth per `docs/features/ELEVENLABS_API_MOBILE.md` (`X-API-Key` + `uid` for limits). Document in a short **`AI_IMAGE_API_MOBILE.md`** when implemented.

6. **MCP** — extend tools so agents can request image generation with **prompt + allowed `model_id`**, optional **reference_media_id** (user-owned media), optional **`style_prompt_id`** (library id resolved server-side), subject to same server checks and quotas; update `docs/public/MCP.md` tool tables.

7. **Style prompt library** — gated **`GET`** catalog with **`tags[]`** and optional **`?tag=`** filter; merge **`style_prompt_id` → snippet** on generate; store curated data in repo or Firestore.

## Risks and constraints

- **Cost:** Image models vary widely in price and latency; queue wait can exceed TTS. Consider timeouts and user-facing progress (fal `onQueueUpdate` can be logged server-side).
- **Safety:** User-supplied prompts may violate fal or platform policies; return clear errors and avoid leaking raw provider errors to clients.
- **Style library IP:** Use **only** Deckbase-original or properly licensed prompt text; do not copy from external marketplaces.
- **Dimensions / crop:** Image blocks support **display aspect** and crop; generated assets may need **default dimensions** matching template expectations or post-process resize (optional follow-up).
